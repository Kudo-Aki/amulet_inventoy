# GitHub Pages 公開ガイド

このガイドでは、お守り在庫管理アプリをGitHub Pagesで無料公開する方法を、初心者の方にもわかりやすく説明します。

---

## 全体の流れ

```
1. GitHubアカウントを作成（5分）
      ↓
2. 新しいリポジトリを作成（2分）
      ↓
3. アプリのファイルをアップロード（3分）
      ↓
4. GitHub Pagesを有効化（2分）
      ↓
5. 公開URLを確認（1分）
      ↓
完了！ 誰でもアクセスできるようになります
```

---

## STEP 1：GitHubアカウントを作成する

### 1-1. GitHubにアクセス

ブラウザで以下のURLを開きます：

```
https://github.com
```

### 1-2. Sign up をクリック

画面右上の「**Sign up**」ボタンをクリックします。

### 1-3. 必要情報を入力

| 項目 | 入力内容 |
|------|---------|
| Email | メールアドレス（神社のアドレスでOK） |
| Password | パスワード（8文字以上） |
| Username | ユーザー名（英数字、例：jinja-omamori） |

「**Create account**」をクリックします。

### 1-4. メール認証

登録したメールアドレスに確認メールが届きます。
メール内の「**Verify email address**」をクリックして認証を完了します。

---

## STEP 2：新しいリポジトリを作成する

### 2-1. リポジトリ作成画面を開く

GitHubにログインした状態で、画面右上の「**+**」マークをクリックし、「**New repository**」を選択します。

### 2-2. リポジトリ情報を入力

| 項目 | 入力内容 |
|------|---------|
| Repository name | `omamori-app`（任意の名前でOK） |
| Description | 「お守り在庫管理アプリ」（任意） |
| Public / Private | **Public** を選択（※重要） |
| Add a README file | チェックを**入れない** |

「**Create repository**」をクリックします。

> **注意**：GitHub Pagesを無料で使うには「Public」にする必要があります。

---

## STEP 3：アプリのファイルをアップロードする

### 3-1. アップロード画面を開く

リポジトリが作成されると、以下のような画面が表示されます。
「**uploading an existing file**」というリンクをクリックします。

### 3-2. ファイルをドラッグ＆ドロップ

1. zipファイルを解凍して、`app`フォルダを開きます
2. `app`フォルダの**中身すべて**を選択します
   - `index.html`
   - `css`フォルダ
   - `js`フォルダ
   - `data`フォルダ
   - `sample`フォルダ
   - その他のファイル

3. 選択したファイルをGitHubの画面にドラッグ＆ドロップします

> **重要**：`app`フォルダ自体ではなく、**中身のファイル**をアップロードしてください。

### 3-3. コミット（保存）する

画面下部の「**Commit changes**」ボタンをクリックします。

アップロードが完了すると、ファイル一覧が表示されます。
`index.html`が一覧に表示されていればOKです。

---

## STEP 4：GitHub Pagesを有効化する

### 4-1. Settings を開く

リポジトリ画面の上部にある「**Settings**」タブをクリックします。

### 4-2. Pages を開く

左側のメニューから「**Pages**」をクリックします。

### 4-3. Source を設定

「**Source**」セクションで以下を設定します：

| 項目 | 設定値 |
|------|--------|
| Source | **Deploy from a branch** |
| Branch | **main** |
| Folder | **/ (root)** |

「**Save**」ボタンをクリックします。

### 4-4. 公開を待つ

設定後、1〜2分待つと公開が完了します。
画面上部に緑色のチェックマークと公開URLが表示されます。

---

## STEP 5：公開URLを確認する

### 5-1. URLの形式

公開URLは以下の形式になります：

```
https://[ユーザー名].github.io/[リポジトリ名]/
```

例：
```
https://jinja-omamori.github.io/omamori-app/
```

### 5-2. 動作確認

このURLをブラウザで開き、アプリが正常に表示されることを確認します。

## よくある質問

### Q: Publicにすると誰でも見られますか？

**A:** はい、URLを知っている人は誰でもアクセスできます。ただし、URLを公開しなければ、実質的に見つけられることはほとんどありません。また、このアプリはデータをサーバーに保存しないため、情報漏洩のリスクはありません。

### Q: 後からファイルを更新できますか？

**A:** はい、同じ手順でファイルをアップロードすれば上書きされます。

### Q: 費用はかかりますか？

**A:** いいえ、GitHub Pagesは完全無料です。クレジットカードの登録も不要です。

### Q: Privateリポジトリでも使えますか？

**A:** 有料プラン（GitHub Pro、月$4）であれば、Privateリポジトリでも GitHub Pages を使えます。

## トラブルシューティング

### ページが表示されない（404エラー）

| 原因 | 対処法 |
|------|--------|
| まだ公開処理中 | 5分ほど待ってから再度アクセス |
| index.htmlがルートにない | `app`フォルダではなく中身をアップロード |
| Branchの設定ミス | Settings > Pages で「main」を選択 |

### ファイルがアップロードできない

| 原因 | 対処法 |
|------|--------|
| ファイルサイズが大きい | 25MB以上のファイルは分割が必要 |
| ファイル数が多い | 100ファイル以下に分けてアップロード |
